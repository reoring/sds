# po-handover — PO session の世代交代(送り出し側)

[English](SKILL.md) | **日本語**(参考訳 — 実行時に読み込まれる正本は英語版)

> PO session の引き継ぎファイルを書いて、安全に session を入れ替える —
> worker の「session は使い捨て」規律の PO 版。context 使用率が 50〜60% に
> 達したとき、または gate が閉じた・wave が終わったなどの自然な区切りで、
> **判断が鈍る前に**実行する。新 session は /po-resume で引き継ぐ。

**原則: PO の状態は正本文書・台帳・inbox にあるべきで、context にしか無い
状態はそれ自体が欠陥。** 引き継ぎファイルに書くのは「地図 + 再測定できない
進行中の差分」だけ — 記憶のダンプではない。混乱してからではなく、混乱する前に回す。

## いつ回すか

- context 使用率が **50〜60%** に達したとき(60% を超えた混乱は実測済みの症状)
- 自然な区切り: gate が1つ閉じた、wave が完了した
- 判断の質が落ちた自覚があるとき(遅いが、やらないよりはいい)

refresh はタダだ。惜しまない — worker の「有料のモデル昇格より先に、無料の
立て直しを」とまったく同じ考え方。

## 手順

### 1. 引き継ぎファイルを書く

固定パス: `<PO運用dir>/handover/<space>.md`。テンプレート:

```markdown
# PO handover: <space>
- 交代日時: <date time> / 旧 session の役割: <一行>

## 進行中(再測定できないものだけ)
- GO 待ちの gate: <どれが・誰の承認待ちか。無ければ「なし」>
- 待っている verdict / receipt: <誰から・どのパスに届く予定か>
- lane・オーナーへの約束: <「〜したら報告する」類。無ければ「なし」>
- 正本にまだ書いていない直近の指示: <内容と出所。存在すること自体が黄信号>

## Monitor 再武装リスト(Monitor は session と共に死ぬ — 必須)
- <各 Monitor の説明と、引数まで含めた実行コマンド全部>
- (無ければ「なし」と明記)

## inbox の基準線
- <inbox dir>: <このファイルまで処理済み(ls -t の先頭 + 時刻)>

## 正本へのポインタ(中身は写さない — 写しは第二の正本になる)
- 運用文書: <パス>
- 台帳: <モデル台帳 / 判定台帳のパス>
- フロー正本: <dev-flow 文書 / 設計原則のパス>
```

### 2. 自己検査(fail-closed)

- [ ] 「上で話したとおり」など、**このファイルの外への参照がゼロ**
- [ ] 初見の PO が「このファイル + live 実測」だけで安全に動けるか
- [ ] 進行中の各項目に「次にやること」が1行付いているか
- [ ] 正本に書くべきなのに context にしか無かった事実に気づいたら、
      引き継ぎにではなく**正本側にいま書く**(引き継ぎはポインタだけ)

**書きにくいと感じたら、それが検知器** — 状態を context に溜め込んでいた証拠だ。
次の世代の規律: 指示・裁定・gate の状態は、受けたその瞬間に正本へ書く。

### 3. 自分の後始末

- 自分が立てた Monitor / background task を止める(TaskList → TaskStop)。
  止めずに session が終わると、monitor が黙って死んでイベントを取りこぼす
  (本番で実測済み)
- 未送信の連絡があれば送り切ってから交代する

### 4. 起動の一行を出力する

最後のメッセージの末尾に、オーナーが新 session に貼るだけの一行を出す:

```
/po-resume <PO運用dir>/handover/<space>.md
```

## やらないことリスト

- 正本の中身を引き継ぎに転記しない(ポインタのみ)
- 「たぶんこうだったはず」を書かない — 不確かなら「未確認」と書く
- 60% を大きく超えてから慌てて書かない(混乱した頭で描いた地図は歪んでいる)
