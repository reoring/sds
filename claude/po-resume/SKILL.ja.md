# po-resume — PO session の世代交代(受け取り側)

[English](SKILL.md) | **日本語**(参考訳 — 実行時に読み込まれる正本は英語版)

> /po-handover の引き継ぎファイルから PO session を再開する。引き継ぎは
> 「地図」として読み、状態は live 実測で組み立て直し、Monitor を再武装し、
> PO の標準道具箱(dev-flow / issue-lane / herdr-event-watch + fleet の
> 台帳・連絡規約)を意識に載せてから業務に入る。引数は引き継ぎファイルのパス。

**原則: 引き継ぎは地図であって、記憶ではない。** 受け取るのはポインタと
進行中リストだけ — 前の session の思い込みは相続しない。すべて live で測り直す。
以下の順番を崩さないこと。

## 手順(この順で)

### 1. 引き継ぎを読む

引数のパス(省略時は `<PO運用dir>/handover/<space>.md`)。取り出すのは:
①進行中(gate / verdict 待ち / 約束)②Monitor 再武装リスト ③inbox の基準線
④正本へのポインタ。

### 2. 正本を読む(ポインタの先)

運用文書 → 台帳(モデル台帳・判定台帳)→ フロー正本(dev-flow 文書・設計原則)。
大きいファイルは必要なセクションだけ読む。

### 3. 標準道具箱を意識に載せる(必須 — 飛ばさない)

新しい session はスキルの存在を「知って」いても、「使う判断」に載っていない
ことが多い。以下をひとつずつ明示的に確認し、再開報告に「装備確認済み」と書く:

| スキル | PO 業務での使いどころ |
|---|---|
| **dev-flow** | 工程管理と gate 判定 — 特に工程5の hollow-green 防御(実装者の test diff ゼロ / held-out test / 本物の境界での green) |
| **issue-lane** | lane の割当・撤収・監査。1 issue = 1 lane、二層ラベル(tab × pane)、issue が閉じたら lane も畳む、劣化 lane の世代交代(有料 boost より先に無料 refresh) |
| **herdr-event-watch** | 監視の再武装(手順5)とイベント駆動の見守り。INBOX が主信号、LANE は保険、「通知は『見に行け』であって『信じろ』ではない」 |
| fleet のモデル台帳規約 | boost / 降格の記帳。行き詰まりの4兆候(同一失敗の反復 / 自己申告 / 無進捗 Working / **宣言停止** = 計画を語るだけで diff ゼロの turn) |
| fleet の連絡規約 | pane への確実な送信(composer に打ち込んだだけで未送信、の罠に注意)。宛先は必ず pane ID |
| fleet の巡回規約 | イベント監視が死んだときの、時間ベースの保険 |

### 4. live で測り直す(引き継ぎを信じない)

- `herdr workspace list` → 担当 space の `herdr tab list` × `herdr pane list` を
  tab_id で突き合わせ(issue-lane の三点照合と同じ形)
- 進行中 issue の tracker 上の現在状態(引き継ぎが書かれた後の動きを検出する)
- inbox を基準線から先へ sweep: 記録された filename より新しいものは全部
  処理キューへ
- モデル台帳の有効な記載と、pane フッターの実測を突き合わせ(無断 boost の検出)

### 5. Monitor を再武装する

引き継ぎのリストどおりに立て直す。**再武装した直後に必ず手で sweep する**
(監視が止まっていた間の到着分は通知されない — 既知の罠)。リストが「なし」でも、
space に動いている lane がいるなら監視ゼロで運用しない — 最低1本立てるか、
巡回 cron が生きていることを確認する。

### 6. 世代交代を記録する

- 引き継ぎファイルを `<PO運用dir>/handover/archive/<space>-<日時>.md` へ移す
- inbox / 通知チャンネルに一言:「PO resumed(<space>、進行中 N 件引き受け)」—
  いま誰が PO なのかを消えない形で fleet に残す(PO 取り違えの予防)

### 7. 再開報告(最後のメッセージ)

オーナーに1本で: ①装備確認済みの道具箱 ②再武装した Monitor ③sweep で
見つかった新着 ④引き受けた進行中リストと、各項目の次アクション
⑤引き継ぎと live 実測が食い違った点(あれば — 前 session の思い違いは正直に)。

## やらないことリスト

- 引き継ぎの記述を、測り直さずに事実として扱わない
- 手順3を「読んだからいい」で飛ばさない — 60% 混乱の再発を防げるかは、
  道具箱が「知っている」でなく「載っている」かで決まる
- 再開直後に重い裁定をしない — 手順1〜6 を終えてから業務に入る
